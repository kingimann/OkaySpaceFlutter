  APP-SIDE PATH/METHOD FIXES (backend route exists under another name):
    [FIXED 2026-06-13 in admin_service.dart]

      /admin/reset/analytics (slash, not hyphen).

      has POST /admin/badges + DELETE /admin/badges/{id}. Add a GET to list,
      or remove the app's badge-list read.

§2b is intentionally left open as backend hygiene. The app now robustly parses
ALL the shapes the backend emits — top-level {"error":{code,message}}, a
business-error {"detail":{code,message}}, a plain {"detail":"msg"}, and the
FastAPI 422 {"detail":[…]} list — so the dual shape is no longer breaking
(fixed app-side 2026-06-13: ApiException previously dropped the code/message of
the {"detail":{code,message}} business errors — rate limits, self-transfer,
wrong security answer, invite-required — and showed a generic status message).
Converging on one envelope is now purely an SDK-friendliness cleanup: pick one
shape, declare it as the 4xx/5xx response_model app-wide, and publish the
"code" values. Best done as its own coordinated backend pass.
