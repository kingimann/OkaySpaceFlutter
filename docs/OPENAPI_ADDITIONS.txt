NEWLY WIRED — this update (spec shipped → app now consumes)
--------------------------------------------------------------------------------
✔ Engagement detail
    GET /posts/{id}/likers     → [PublicUser]
    GET /posts/{id}/reposters  → [PublicUser]
    GET /posts/{id}/viewers    → {count, unique, viewers:[ViewerOut]}
    GET /posts/{id}/analytics  → {impressions, unique_viewers, clicks,
                                  reactions_total, comments, reposts, quotes,
                                  bookmarks, interactions, engagement_rate}
    App: tappable like/repost counts open "who liked / reposted" sheets; the
    post author gets a read-only "View insights" sheet (analytics + recent
    viewers).
✔ Post drafts (server-side, replacing local-only multi-drafts)
    GET    /drafts                → [Draft{id,payload,created_at,updated_at}]
    POST   /drafts   {payload}    → Draft
    PATCH  /drafts/{id} {payload} → Draft
    DELETE /drafts/{id}           → {ok}
    App: composer "Save to drafts" + the Drafts sheet now read/write the
    server; local secure-storage remains a transparent offline fallback.
✔ Subscription tiers
    GET /subscription-tiers      → {tiers:[...]}   (+ tier on /users/{id}/subscribe)
    App: subscribing to a creator shows a tier picker when tiers exist; falls
    back to a plain subscribe when the list is empty.
✔ Place reviews
    GET    /reviews?place_key=…   → [Review]
    POST   /reviews  {place_key, place_name, rating, longitude, latitude, text}
    DELETE /reviews/{id}          → {ok}
    App: a saved place on the map opens a Reviews screen (average rating +
    list, write/delete your own review).

REMAINING NICE-TO-HAVES (optional — app works without them)
--------------------------------------------------------------------------------
[ ] §3b  kind field on Post / PostCreate ("post" | "reel" | "video"). The app
         still infers a "video" from a titled video post and a "reel" from an
         untitled one; an explicit kind removes that heuristic.
[ ] §3c  GET /feed/videos (landscape/titled videos only). Still absent — the
         Videos screen merges /feed/reels + /reels/popular and keeps the
         titled ones client-side.
[ ] §5   Spec hygiene (makes the whole spec SDK-generatable):
         - Declare response_model on the money/payout/stripe routes that still
           have an empty 200 schema, so the app can stop key-probing
           (client_secret ?? clientSecret ?? secret, etc).
         - One error envelope: responses still carry BOTH {"error":{...}} and
           {"detail":{...}} — pick one and document the "code" values.
         - Auth before body-validation on protected routes (some 422 before
           checking the token; should 401 first).
         - Timestamps ISO-8601 UTC with trailing Z; IDs always strings in JSON.
